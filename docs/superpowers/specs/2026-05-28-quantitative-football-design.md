
# 量化足球系统 —— 设计文档

> 版本: v1.0 | 日期: 2026-05-28 | 状态: 待评审

## 1. 项目概述

### 1.1 目标

构建一个**数据驱动的足球分析与投注决策辅助系统**，覆盖世界杯、五大联赛（英超、西甲、德甲、意甲、法甲）和欧冠。
通过多因子分析 + 蒙特卡洛模拟 + 策略引擎，输出买入建议。系统不自动下单，用户手动执行。

### 1.2 核心原则

- **数据真实优先**：优先使用免费官方 API，失败降级 CLI/缓存
- **信息渐进**：世界杯信息盲区通过"数据可信度"机制处理，赛后自动校准
- **可编辑可持久化**：因子权重和方案可前端调整、多方案保存
- **LLM 增强非替代**：大模型做情报解读和定性分析，定量模型为主体
- **轻量化**：单页面 Web + SQLite，一键启动

### 1.3 技术栈

| 层 | 技术 |
|---|---|
| 后端 | Python 3.12+, FastAPI |
| 数据库 | SQLite + SQLAlchemy |
| 前端 | 单页 HTML/CSS/JS（原生或 Alpine.js） |
| LLM | OpenAI 兼容接口（GPT-4o-mini 推荐） |
| 数据抓取 | httpx + football-data.org API + CLI 工具 |
| 模拟 | 自建泊松分布蒙特卡洛引擎 |

## 4. 核心数据流

```
Step 1: 数据获取
    ├── 比赛日程抓取 (football-data.org / FIFA API)
    ├── 球队 & 球员基础数据
    ├── 新闻多源聚合 & 交叉验证
    ├── 首发阵容预测 (基于近期首发 + LLM)
    ├── 赔率数据抓取
    └── 清洗入库 (SQLite)

Step 2: 因子分析
    ├── 读取当前方案的因子权重
    ├── 计算每个因子的得分
    ├── 应用置信度系数 (世界杯首场自动降权)
    ├── 加权求和 → 实力差值 λ
    └── 输出两队的期望进球率

Step 3: 蒙特卡洛模拟
    ├── 泊松(λ_home) → 主队进球
    ├── 泊松(λ_away) → 客队进球
    ├── 循环 10000 次
    └── 输出: 胜/平/负概率, 比分分布, 总进球期望

Step 4: 结果总结
    ├── 模拟结果汇总
    ├── 对比赔率隐含概率
    ├── LLM 生成自然语言总结
    └── 前端图表展示

Step 5: 风险分析
    ├── 模拟结果方差 / 标准差
    ├── 最大回撤估算
    ├── 凯利公式仓位计算
    └── 风险等级标签 (低/中/高)

Step 6: 买入策略
    ├── 期望值 = 模拟概率 x 赔率 - 1
    ├── 期望值 > 0 → 有价值
    ├── 凯利公式 → 建议仓位
    └── 前端展示: 建议买入/观望/回避
```

## 5. 因子体系

### 5.1 因子分类（L1-L5）

| 层级 | 名称 | 描述 |
|---|---|---|
| L1 | 球队基本面 | 战绩、阵容、实力等基础指标 |
| L2 | 战术风格 | 打法、阵型、风格克制 |
| L3 | 环境因素 | 天气、裁判、旅途等外部条件 |
| L4 | 心理/战意 | 德比、士气、杯赛动机 |
| L5 | 市场信号 | 赔率、资金流向 |

### 5.2 默认因子列表（19 个）

| 层级 | 因子 | 默认权重 | 说明 |
|---|---|---|---|
| L1 | 近期战绩 | 20% | 近 5/10 场胜率、进球、失球 |
| L1 | 主客场分离 | 10% | 主场 vs 客场表现差异 |
| L1 | 阵容完整度 | 12% | 伤病/停赛，对比理论最强 11 人 |
| L1 | 射手状态 | 8% | 主力射手近期进球效率 |
| L1 | 配合默契度 | 5% | 预测首发球员国家队同场次数 |
| L1 | 教练稳定性 | 3% | 教练在任时长、体系成熟度 |
| L2 | 控球率倾向 | 5% | 控球型 vs 放弃控球型 |
| L2 | 进攻方式 | 6% | 边路传中 / 中路渗透 / 快攻反击 |
| L2 | 防守风格 | 6% | 高位压迫 / 中位拦截 / 低位大巴 |
| L2 | 风格克制 | 8% | 高位防线 vs 快马，大巴 vs 远射 |
| L3 | 天气 | 3% | 雨战、大风、高温对战术的影响 |
| L3 | 裁判风格 | 7% | 出牌率、判罚尺度 vs 球队风格 |
| L3 | 旅途疲劳 | 3% | 飞行距离、休息天数 |
| L3 | 赛程密度 | 4% | 一周双赛/三赛的轮换影响 |
| L4 | 战意 | 8% | 杯赛/保级/争冠动力 |
| L4 | 历史交锋 | 4% | H2H 心理优势 |
| L4 | 近期士气 | 5% | 连胜势头 vs 连败低迷 |
| L4 | 大赛经验 | 4% | 世界杯/欧冠决赛圈经验 |
| L5 | 赔率变化 | 5% | 初盘→即时盘走势 |

**设计原则：**
- L1 基本面占 58%，是模型根基
- 裁判因子 7%——保守值，遇裁判 vs 球队风格冲突时手动调高
- 市场信号仅 5%，保持独立判断
- 权重总和分析引擎内部归一化处理

### 5.3 世界杯信息渐进

| 比赛阶段 | 信息状况 | 置信度系数 | 影响 |
|---|---|---|---|
| 小组赛 MD1 | 仅预选赛/友谊赛数据 | 0.5-0.7 | 模拟方差大，分布宽 |
| 小组赛 MD2 | 有 1 场正赛数据 | 0.7-0.85 | 方差收窄 |
| 小组赛 MD3+ | 数据充足 | 0.85-1.0 | 接近联赛水平 |
| 淘汰赛 | 数据充足，战意放大 | 0.85-1.0 | 战意因子权重大 |

系统自动感知比赛阶段，调整所有因子的置信度系数。

## 7. 新闻交叉验证系统

### 7.1 来源分级

| Tier | 类型 | 示例 | 信任度 |
|---|---|---|---|
| 1 | 官方 | 球队官网、FIFA、发布会原文 | 100% |
| 2 | 一线权威 | BBC、天空体育、队报、踢球者 | 75% |
| 3 | 跟队记者 | Fabrizio Romano、各队跟队 | 60% |
| 4 | 普通媒体 | 太阳报、每日体育报 | 35% |
| 5 | 小道消息 | Twitter、Reddit | 15% |

### 7.2 验证流程

```
多源聚合 → 事件聚类 → 提取各方说法 → 共识度计算
                                        ↓
                    共识度 >= 70% → 高信(采纳)
                    共识度 40-70% → 中信(展示分歧)
                    共识度 < 40% → 低信/烟雾弹(不纳入因子)
```

### 7.3 烟雾弹识别（LLM 辅助）

- 仅 Tier 4-5 报道，高等级来源沉默 → 假消息
- 来源前后矛盾 → 标记不可靠
- 赛前 24h 放出核心缺阵 → 战术烟雾弹模式
- 对手阵营媒体单方面唱衰 → 心理战
- 教练发言与训练照/视频不一致 → 实锤

## 9. 模拟引擎

### 9.1 模型

泊松分布进球模型：

```
λ_home = baseline_home x exp(sum(weight_i x factor_score_i))
λ_away = baseline_away x exp(-sum(weight_i x factor_score_i))

P(home_goals=k) = (λ_home^k x e^(-λ_home)) / k!
P(away_goals=k) = (λ_away^k x e^(-λ_away)) / k!
```

方差随信息不确定性增大：

```
σ^2 = λ x (1 + uncertainty_coefficient)
uncertainty_coefficient = sum(weight_i x (1 - confidence_i))
```

### 9.2 输出

- 胜/平/负百分比
- 比分分布 Top 5
- 总进球期望
- 主客队进球分布图
- 模拟方差

## 10. 策略引擎

### 10.1 风险分析

- 模拟胜率 vs 赔率隐含胜率对比
- 期望值 = 模拟概率 x 赔率 - 1
- 模拟方差（分布宽度 → 不确定性风险）

### 10.2 买入策略

**凯利公式：**

```
f = (b x p - q) / b

其中: b = 赔率-1, p = 模拟胜率, q = 1-p
建议仓位 = min(f x 风险管理系数, 最大仓位上限)
```

**输出：**
- 期望值正负
- 建议：买入/观望/回避
- 建议仓位比例
